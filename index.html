Introduction

The microstructural and chemical characterization relies mostly on the interaction of a material with a probe.

This probe is usually visible light, X-rays or a beam of high energy electrons. The resultant signal will then be collected, processed and interpreted.

10

Introduction

  • Microscopy: allows looking at objects not visible for the naked eye.
  • Limitation: the smallest distinguishable features are determined by the wavelength of visual light (0.4 - 0.7 μ m)
  • By changing the source of illumination to a radiation probe with a smaller wavelength, smaller details can be observed.

  • Since electrons can be accelerated up to a velocity of almost the speed of light => possible to create electrons with a wavelength ≈ 0.1 nm => possible to go down the atomic and molecular scale of matter.
  • In electron microscopes: a high voltage is used to accelerate electrons emitted from a heated filament, which can then be focused using magnetic lenses onto a sample; resolution in these devices limited by lenses.

  • X-rays typically have a wavelength below 10 nm => in practice resolution is limited because of other reasons (focusing devices).
  • Their penetrating power is largely dependent on their E, which is directly related to their wavelength:
    • The energy of an X-ray is reversely proportional to its wavelength and is calculated by the equation "E= hc/ lambda ", where h is Planck constant, c is speed of light and lambda is the wavelength. X-ray energy is typically given in electronvolts ( eV ).

BIT DEPTH

= determined by the number of bits used to define each pixel.

The greater the bit depth, the greater the number of tones (grayscale or color) that can be represented.

Digital images may be produced in black and white ( bitonal ), grayscale, or color.

45

BIT DEPTH

A bitonal image is represented by pixels consisting of 1 bit each, which can represent two tones (typically black and white), using the values 0 for black and 1 for white or vice versa.

46

BIT DEPTH

A grayscale image is composed of pixels represented by multiple bits of information, typically ranging from 2 to 8 bits or more.

47

BIT DEPTH

  • Example : In a 2-bit image:
  • 4 possible combinations: 00, 01, 10, and 11.
  • If "00" = black , and "11" = white: then "01" equals dark gray and "10" equals light gray.
  • bit depth = 2
  • the number of tones that can be represented is 2 2 or 4.
  • At 8 bits, 256 ( 2 8 ) different tones can be assigned to each pixel.

BIT DEPTH

A colour image is typically represented by a bit depth ranging from 8 to 24 or higher.

With a 24-bit image, the bits are often divided into 3 groupings : 8 for red, 8 for green, and 8 for blue. Combinations of those bits are used to represent other colours.

A 24-bit image offers 16.7 million ( 2 24 ) colour values.

Binary calculations for the number of tones represented by common bit depths:

1 bit (2 1 ) = 2 tones

2 bits (2 2 ) = 4 tones

3 bits (2 3 ) = 8 tones

8 bits (2 8 ) = 256 tones

16 bits (2 16 ) = 65,536 tones

24 bits (2 24 ) = 16.7 million tones

DYNAMIC RANGE

= the range of tonal difference between the lightest light and darkest dark of an image.

The higher the dynamic range, the more potential shades can be represented, although the dynamic range does not automatically correlate to the number of tones reproduced.

Dynamic range also describes a digital system's ability to reproduce tonal information. This capability is most important for continuous-tone documents that exhibit smoothly varying tones, and for photographs it may be the single most important aspect of image quality.

FILE SIZE

= calculated by multiplying the surface area of a document (height x width) to be captured by the bit depth and the dpi 2 . Because image file size is represented in bytes, which are made up of 8 bits, divide this figure by 8.

Formula 1 for File Size:

  • File Size = (height x width x bit depth x dpi 2 ) / 8
60

FILE SIZE

If the pixel dimensions are given, multiply them by each other and the bit depth to determine the number of bits in an image file. For instance, if a 24-bit image is captured with a digital camera with pixel dimensions of 2,048 x 3,072, then the file size equals (2048 x 3072 x 24)/8, or 18,874,368 bytes.

Formula 2 for File Size

  • File Size = (pixel dimensions x bit depth) / 8

FILE SIZE

File size naming convention: Because digital images often result in very large files, the number of bytes is usually represented in increments of 2 10 (1,024) or more:

1 Kilobyte (KB) = 1,024 bytes

1 Megabyte (MB) = 1,024 KB

1 Gigabyte (GB) = 1,024 MB

1 Terabyte (TB) = 1,024 GB

COMPRESSION

= used to reduce image file size for storage, processing, and transmission. The file size for digital images can be quite large, taxing the computing and networking capabilities of many systems.

All compression techniques abbreviate the string of binary code in an uncompressed image to a form of mathematical shorthand, based on complex algorithms.

There are several compression techniques available.

63

Compression

Compression schemes: characterized as either lossless (= “ verliesvrij ”) or lossy (= “ kwaliteitsverlies “)

Lossless schemes: abbreviate binary code without discarding any information => when image is "decompressed" it is bit for bit identical to original; often used with bitonal scanning of textual material.

Lossy schemes: such as JPEG, utilize a means for averaging or discarding the least significant information, based on an understanding of visual perception. However, it may be extremely difficult to detect the effects of lossy compression; typically used with tonal images.

Explanation of File Formats

GIF:

  • Graphics Interchange Format
  • lossless file format
  • preferred for graphics, because it keeps edges and lines sharp.
  • limited to 256 or fewer colours
  • not recommended for photographs

PNG:

  • Portable Network Graphics format
  • an open source substitute for GIFs.
  • provides a higher lossless compression rate than GIFs.
  • provides a useful format for the storage of images during intermediate stages of editing.

Image quality

The photography: a crucial step for successful image analysis, which requires a high quality image with which to work.

=>The digital camera must provide resolution appropriate for the type of image and magnifications of interest;

Most analyses require uncompressed file formats such as TIFF to prevent loss of image quality.

What is considered a good quality image can vary.

  • For published descriptions of a material: an image that closely resembles what one sees when looking in the microscope is often desirable.
  • For image analysis: may require enhancement of certain features, exaggeration of contrast, etc.
  • Multiple versions of the same image may be produced during preprocessing, during various phases of analysis, and as publications are prepared; the original uncompressed image must always be retained!
71

IMAGE ENHANCEMENT OR PREPROCESSING

often necessary to enhance features of interest in order to perform an analysis.

When a new project is initiated, identify a reliable protocol for enhancement. The same procedure can be applied routinely to similar materials undergoing the same analysis, and often routines can be set up to perform image enhancement or preprocessing steps automatically.

Includes changing contrast, brightness, applying filters, etc…

Always keep original image!
